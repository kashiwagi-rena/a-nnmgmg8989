## 1. なぜデータ構造は重要
> 高品質コードには別の尺度、コード性能（効率性）がある。

コード性能って、なんだぁ？
直感的にはわかるけど、ちゃんとした説明ができない。

>  配列（array）はコンピュータサイエンスで最も基本的なデータ構造だ。
配列はデータ要素のリストだ。

配列もデータ構造だよね、知ってるもん！
とても身近だった

> データ構造の性能理解には、コードがデータ構造に関わる一般的な方式を分析する必要がある<br>
* 読み：データ構造の特定の位置にある何かを参照することを指す<br>
* 探索：ある値がデータ構造中にあるか探すことを指す<br>
* 挿入：データ構造に新たな値を追加することを指す<br>
* 削除：データ構造から値を取り除くことを指す<br>

暗記！

> **処理の速度を測るとき、時間ではなく、ステップ数で測る**

心に刻みました
確かに、時間はコードを見てもわかんないけどステップ数はわかる！
わかりやすい〜

> ステップ数の測定が処理速度の鍵となる

鍵となるだけで、絶対ではないのもちゃんと認識する
**時間計算量の測定**のことなので、空間計算量も存在する

> 集合：重複値を含まないデータ構造

配列と集合をPCから見て、扱う際の話が載っている。集合はめんどいので、私がPCなら、「集合がきたよ・・・めんど・・・」って思っている。
ものは使いようなので、ちゃんと使えるようになりたい。（いつも配列・集合ってどこで使うんだ？）

##  次際にコードを動かしてみた
以下、本を参考に書いたコードである。
同じ処理を行うコードだが、2番目のコードの方が早いと思う。
```
const print_numbers_version_one = () => {
    let number = 2
    while (number <= 100) {
        if(number % 2 === 0) {
            console.log(number);
        }
        number += 1
    }
}

const print_numbers_version_two = () => {
    let number = 2
    while (number <= 100) {
        console.log(number);
        number += 2
    }
}

```

実際に動かした実行結果は、
```
=== 実行時間比較 ===

4
6
~~~

100
Version 1 (偶数チェック版): 0.7108 ms

Version 2 (2ずつ増加版): 0.2905 ms

--- 比較結果 ---
Version 2 (2ずつ増加版)の方が2.45倍高速
```

2つ目のコードの方が、2.45倍時間的に早かった
ではなぜ、「2番目のコードの方が早いのか」方が早いのだろう？
ステップ数がprint_numbers_version_twoの方が半分なので、早い
わかりやすい！
割り算もif文も比較演算もなくなるため、2倍以上早くなっている！
