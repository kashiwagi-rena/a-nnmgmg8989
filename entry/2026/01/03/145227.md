## コードをO表記で速くする

O表記**で**、早くするのはむりなのでは？<br>
表記でしかないのに、どゆこと？という感じ<br>

> O表記は性能を表す重要な道具だ。
本章では、実際の問題を解くコードを書いてO表記でアルゴリズムの性能測定をする。それから、アルゴリズムを改良して効率を上げられるか試す

そういうことか！<br>
性能を表す道具なので、自分のコードの性能の見直しと再構築の機会になるのか<br>

バブルソートは最初に覚えたソートで理解するのは簡単だけどO(N2)なのか・・・めちゃくちゃ非効率<br>

O(N2)：二条時間<br>

> アルゴリズムのループの中にループがあると、(常にではないが)しばしばO(N2)になる。入れ子ループがあるならO(N2)の警告が頭の中で鳴り出すべきだ。O(N2)は遅いアルゴリズムだから、少し時間を掛けてもより速いアルゴリズムがないかをどうかを調べるべきだ。

入れ子ループは確かに遅い(仕事で出会ったことのあるO(N2)のコードはかなり遅く・・・時間がかかりすぎて画面表示ができなかった)<br>
その場合、元のデータがどんなデータなのかを確認し、よりあったアルゴリズムにするといい<br>

> O表記をしっかり理解することで、2つのアルゴリズムがあったときに遅いコードを見分けて速い方を選べるようになるのは明らかだ。

AIが出すコードをちゃんと選べるようになる第一歩になってくれそうな予感<br>
効率的で速いコードかどうかをO表記を確認できるようになりたい

##  次際にコードを動かしてみた
下のコードは、O(N2)のコードである。<br>
入れ子ループがある典型的なO(N2)と認識<br>
ステップ回数も多く<br>
```
function hasDuplicateValue(array) {
    let steps = 0;
    for (let i = 0; i < array.length; i++) {
        for (let j = 0; j < array.length; j++) {
            steps++;
            if (i !== j && array[i] === array[j]) {
                return true;
            }
        }
    }
    console.log(steps);
    return false;
}

console.log(hasDuplicateValue([1, 4, 5, 2, 9]));
```
下のコードは、O(N)のコードである<br>
空配列の使い方がよく、覚えたいところ<br>
少しの配列で、桁数も多くない場合に使えそう<br>
万能的に使えないのが傷・・・
```
function hasDuplicateValue(array: number[]): boolean {
    let steps = 0;
    let existingNumbers: { [key: number]: number } = {};
    for (let i = 0; i < array.length; i++) {
        steps++;
        if (existingNumbers[array[i]] === 1) {
            return true;
        } else {
            existingNumbers[array[i]] = 1;
        }
    }
    console.log(steps);
    return false;
}

console.log(hasDuplicateValue([1, 4, 5, 2, 9]));
```
